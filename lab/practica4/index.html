Redes locales

- Introducción -

En esta práctica, se busca proporcionar una comprensión integral del funcionamiento de una página web mediante la combinación de varios elementos clave. Estos elementos incluyen PHP, un servidor local, un servidor en la nube y la conectividad del módulo ESP8266. PHP es un lenguaje de programación del lado del servidor ampliamente utilizado para el desarrollo web, un servidor local es un entorno de desarrollo que se ejecuta en el propio ordenador del usuario, el servidor en la nube es un servifor remoto accesible a través de internet que aloja y ejecuta aplicaciones web, por útlimo, el módulo ESP8266 es un chip wi-fi de bajo costo y alto rendimiento que utiliza comunmente en proyectos de internet de las coas (IoT).

- Desarrollo -

1. Desarrollar la pagína web

Utilizando bloc de notas se desarrolló una página web con código PHP que permitiera actualizar el valor de un sensor.

2. Despliegue en servidor local utilizando XAMPP

Para este paso, se utilizó XAMPP para colocar la página web en un servidor local. El archivo "index.php" se guardó en la carpeta "htdocs" dentro del directorio de instalación de XAMPP. Luego, se iniciaron los servicios de Apache desde el Panel de Control de XAMPP. Posteriormente, se abrió un navegador web y se accedió a "http://localhost" para verificar el correcto funcionamiento de la página web y su capacidad para actualizar los valores del sensor. También se comprobó el acceso a la página web del servidor.

3. Modificación de datos desde otra computadora mediante conexión con el servidor local

Se realizó la modificación de datos de la página web de PHP desde otra computadora, estableciendo una conexión con el servidor local utilizando la dirección IP correspondiente y la estructura de URL adecuada.

4. Subida de la página web de PHP a un servidor en la nube

Se procedió a subir la página web de PHP a un servidor en la nube y se visualizó en el explorador con la dirección web proporcionada por el servidor. Se verificó su funcionamiento directo desde el explorador.

5. Utilización de NodeMCU y un ejemplo de GET para modificar datos desde el servidor local y en la nube:

Se utilizó el NodeMCU junto con un ejemplo de GET para establecer una conexión con el servidor local y modificar los datos de la página web de PHP. La petición GET utilizada fue del tipo: "http://192.168.0.X/?A=1&B=3". Posteriormente, se repitió el proceso para conectar el NodeMCU a la página web alojada en el servidor en la nube.

  • Conexión

    • - Conclusión -

    Esta práctica proporcionó una comprensión integral del funcionamiento integral de una página web, así como la interacción entre diferentes elementos como PHP, servidores locales, servidores en la nube y dispositivos IoT como NodeMCU. Se pudo verificar cómo estos elementos trabajan en conjunto para permitir la actualización dinámica de datos en una página web, tanto desde el servidor local como desde un servidor en la nube, demostrando la versatilidad y la importancia de estas tecnologías en el desarrollo web actual.
